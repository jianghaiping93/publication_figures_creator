# 设计蓝图（Design Blueprint）

## 项目目标
- 在近 3 年内的顶级期刊（Nature/Science/Cell）论文中，系统搜集公开的 GitHub 代码库。
- 识别并筛选出提供 figure 作图代码的论文与仓库。
- 按图类型对作图代码进行分类、分析与整理。
- 建立“图类型—论文来源—作图代码—示例图”的可检索数据库。
- 构建 1 个 skills 或多智能体体系，能根据用户需求自动选择图类型并复用来源代码，输出统一风格与配色的图。
- 持续同步更新到 GitHub 仓库。

## 范围与边界
- 仅覆盖 Nature/Science/Cell 近 3 年论文。
- 仅收集公开可访问且可复现作图代码的仓库。
- 暂不对论文内容进行学术评价，只聚焦作图代码与图型归类。

## 基本流程（端到端）
1. 论文清单获取：按期刊与年份生成论文列表与 DOI。
2. GitHub 发现：从论文主页/补充材料/README/issue/引用中提取代码仓库。
3. Figure 代码筛选：识别是否包含作图脚本、绘图配置或复现步骤。
4. 图类型分类：按图形类别（如热图、散点、网络图、路径图等）归类。
5. 结构化入库：建立标准字段并入数据库。
6. 统一风格：抽象样式模板（配色/字体/尺寸/注释规范）。
7. 技能/多智能体：封装检索+选型+复用+渲染流程。
8. 版本更新：周期性增量更新与回归复现。

## 关键评估指标（KPI）
- 覆盖率：近 3 年论文中成功检索到仓库的比例。
- 可复现率：能够成功生成 figure 的仓库比例。
- 图类型覆盖度：已覆盖图类型数量与分布。
- 数据库完整度：每条记录字段填充率。
- 统一风格一致性：输出图风格与模板一致的比例。
- 自动化成功率：技能/多智能体一次性成功输出的比例。

## 数据与元数据规范（建议字段）
- 论文信息：期刊、年份、标题、DOI、链接
- 代码仓库：GitHub URL、commit/tag、license
- Figure 信息：图类型、图编号、脚本路径、运行命令
- 依赖与环境：语言、包版本、运行平台
- 产出：示例图路径、生成日志、备注

## 风险与应对
- 论文无公开代码：记录为“无仓库”并保留来源链接。
- 代码不可复现：记录失败原因，建立修复清单。
- 图类型定义不一致：建立统一 taxonomy 并允许多标签。

## 里程碑
- M1：完成论文列表与 GitHub 发现规则。
- M2：完成 figure 代码筛选与基础分类。
- M3：完成数据库雏形与样式模板。
- M4：完成技能/多智能体原型与自动化流程。
- M5：稳定迭代与公开发布。
